# UI STYLE GUIDE — FB BOOKMARKLET TOOLS

Единая дизайн-система для внутренних bookmarklet-скриптов Facebook Ads Manager.

Цель системы — не копировать конкретный скрипт 1:1, а сохранять общий визуальный язык, UX-паттерны и технические константы для разных инструментов: менеджеров пресетов, spend-сканеров, bulk-утилит, cleaners, импортёров, экспортёров и других внутренних тулзов.

## 1. Принцип айдентики

Стиль: compact dark utility UI.

Интерфейс должен ощущаться как:
- внутренний профессиональный инструмент;
- рабочая панель поверх Facebook Ads Manager;
- быстрый utility-виджет, а не полноценный SaaS;
- аккуратный технический интерфейс без лишней декоративности.

Главные качества:
- компактность;
- читаемость;
- скорость восприятия;
- визуальная дисциплина;
- предсказуемая структура;
- минимальный визуальный шум.

Не копировать конкретную сетку, ширину, количество кнопок или порядок блоков из одного скрипта. Для каждого нового инструмента адаптировать структуру под его сценарий.

## 2. Язык интерфейса

Интерфейс всегда на русском языке.

На английском допустимо оставлять:
- названия инструментов;
- API-термины;
- названия сущностей Facebook;
- технические значения, которые нельзя естественно перевести.

На русском должны быть:
- кнопки;
- статусы;
- ошибки;
- подсказки;
- подтверждения;
- названия вкладок;
- лог-сообщения;
- пустые состояния.

Тон интерфейса: спокойный, короткий, рабочий.

## 3. Базовая структура интерфейса

Рекомендуемый порядок блоков:

1. Header: название инструмента, версия, кнопка закрытия.
2. Navigation / Mode Switcher: вкладки, если есть разные сценарии.
3. Main Controls: select, input, checkbox, file picker, account picker.
4. Actions: одна главная кнопка, дополнительные кнопки рядом или ниже.
5. Log / Result: лог обязателен для долгих действий.
6. Note: короткая служебная заметка.

## 4. Layout

### Compact modal
Для большинства маленьких и средних инструментов:
- width: `min(420px, calc(100vw - 24px))`
- max-height: `calc(100vh - 40px)`
- padding: `14px`
- border-radius: `14px`
- centered fixed position
- overflow внутри модалки

### Medium modal
Если инструменту нужно больше полей:
- width: `min(520px, calc(100vw - 24px))`
- padding: `14–16px`

### Large modal
Только для таблиц, предпросмотра данных, batch-редакторов:
- width: `min(720px, calc(100vw - 24px))`
- допускается двухколоночная структура

Не делать large modal без необходимости.

## 5. Цветовая система

Основные цвета:
- panelBg: `#0f1715`
- deepBg / logBg: `#0b1210`
- controlBg: `#121f1b`
- border: `#2b433a`
- inputBorder: `#2f4a40`
- logBorder: `#22372f`
- text: `#e8fff0`
- label: `#c7e0d2`
- muted: `#99b3a6`
- accent: `#4dff8f`
- accentText: `#052012`
- success: `#9bff7d`
- warning: `#ffd27d`
- error: `#ff8f8f`

Акцентный зелёный использовать точечно: заголовок, active tab, primary button, success state, важные статусы. Не заливать весь интерфейс зелёным.

## 6. Типографика

Основной шрифт:
`Inter, "Segoe UI", Arial, sans-serif`

Monospace только для логов и технических значений:
`ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, monospace`

Размеры:
- base text: `13px`
- label: `12px`
- meta/version/note: `11px`
- log: `12px`
- title compact: `28–32px`
- title medium: `24–28px`, если название длинное

Заголовок:
- font-weight: `800`
- color: accent
- line-height: `1.05`

## 7. Компоненты

### Header
Название слева, версия под названием, close справа сверху. Close — текстовый `×` или `X`, без фона.

### Tabs
Использовать только когда есть 2–4 разных режима. Active tab подчёркнут accent border, inactive tab без заливки.

### Fields
Все поля full-width, background `#121f1b`, border `#2f4a40`, radius `9px`, padding `9px`.

### Checkbox
Использовать для опциональных настроек: выбрать все аккаунты, импортировать ширину колонок, использовать встроенный пресет, безопасный режим.

### Primary button
Только для главного действия на экране. Background `#4dff8f`, text `#052012`, font-weight `700`.

### Secondary button
Для дополнительных действий. Background `#121f1b`, text `#e8fff0`, border `#2f4a40`.

### Button groups
Если две равнозначные кнопки — flex row, gap 8px, одинаковая ширина, обе secondary. Если одна главная и одна вторичная — primary сверху/слева, secondary ниже/справа.

### Log
Обязателен для API-запросов, batch-операций, импорта/экспорта, сканирования аккаунтов, обработки файлов.

Стиль: background `#0b1210`, border `#22372f`, radius `10px`, padding `8px`, font-size `12px`, monospace, scrollable.

Формат: `[12:34:56] Сообщение`.

Типы: info, success, warning, error.

## 8. UX-паттерны

- Повторный запуск открытого скрипта закрывает старое окно или заменяет его новым.
- Во время долгого действия блокировать кнопку и менять текст на `Обработка...`, `Загрузка...`, `Импорт...`.
- Подтверждение использовать перед действиями, которые меняют данные.
- Ошибки показывать в лог, alert использовать только для критичных стоп-сценариев.
- Если нужен файл — использовать скрытый input, отмену выбора логировать как warning.
- Если есть встроенный пресет — отдельная secondary-кнопка, не смешивать с импортом JSON.
- Для массовых операций использовать последовательную обработку, задержки, прогресс, success/fail counters.

## 9. Адаптивность

- modal width через `min(...)`
- max-width: `calc(100vw - 24px)`
- max-height: `calc(100vh - 40px)`
- overflow auto
- button groups на узком экране становятся вертикальными
- не использовать fixed pixel width без ограничения

## 10. Запрещено

Не использовать:
- яркие градиенты на фоне;
- glassmorphism;
- большие свечения;
- декоративные иконки без пользы;
- лендинговые hero-блоки;
- карточки ради карточек;
- слишком крупные заголовки;
- английский интерфейс;
- копирование одного скрипта 1:1 в разные инструменты.
