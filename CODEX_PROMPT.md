# CODEX PROMPT — Apply FB Bookmarklet Design System

Изучи:
- `UI_STYLE_GUIDE.md`
- `ui/core/tokens.js`
- `ui/core/typography.css`
- `ui/core/styles.css`

Задача:
привести UI текущего bookmarklet-скрипта к единой дизайн-системе FB Bookmarklet Tools.

Важно:
- не копировать 1:1 внешний вид какого-либо одного скрипта;
- использовать общие паттерны дизайн-системы;
- адаптировать layout под конкретную задачу скрипта;
- не менять бизнес-логику;
- все тексты интерфейса должны быть на русском;
- английский допустим только в названиях инструментов, API-терминах и сущностях Facebook;
- долгие действия должны иметь лог;
- ошибки должны попадать в лог;
- main action должен быть один;
- secondary actions должны быть визуально спокойнее;
- интерфейс должен оставаться compact dark utility tool.

Если у скрипта есть несколько сценариев — использовать вкладки.
Если сценарий один — вкладки не добавлять.
Если данных много — можно использовать medium/large modal, но не делать большой layout без необходимости.
